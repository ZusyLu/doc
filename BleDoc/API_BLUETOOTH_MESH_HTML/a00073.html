Indicate to the stack that a test request has been completed and that the status may be communicated to the Health Client, which made the test request.

Parameters
[in]client_addressAddress of the client model which sent the message
[in]elem_indexServer model element index. Identifies the Server model that received the request as well as the element on which the test is to be performed.
[in]appkey_indexThe application key index to use in encrypting the request.
[in]vendor_idBluetooth vendor ID used in the request
Returns
SL_STATUS_OK if successful. Error code otherwise.
